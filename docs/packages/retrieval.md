# Retrieval 设计

## 文档定位

本文档用于描述 `@rag-sdk/retrieval` 的子包设计。

当前只定义该子包在目标架构中的职责边界，具体接口和实现设计后续在本文档内继续补充。

## 职责

`retrieval` 负责定义检索链路中的阶段协议、阶段间数据传递结构与基础流程编排。

它描述检索前、检索、检索后三个阶段如何协作。

`retrieval` 可以提供不依赖外部系统、不包含业务语义的通用检索处理能力，用于降低调用方组织基础检索流程的成本。

## 阶段模型

检索链路分为三个阶段：

```text
检索前
  → 检索
  → 检索后
```

### 检索前

检索前负责把业务侧传入的原始检索请求整理为可执行的检索查询。

该阶段描述查询文本、查询参数、过滤条件、召回数量等信息如何进入后续检索阶段。

检索前可以包含多个按顺序执行的处理器，例如查询校验、查询标准化、查询改写、过滤条件准备等。

其中，查询改写、查询扩展、查询向量生成等依赖模型或外部服务的能力，只在 `retrieval` 中定义协议，不在 `retrieval` 中实现。

### 检索

检索负责根据检索前阶段产生的查询，从外部索引、向量数据库、搜索引擎或召回服务中获得候选结果。

该阶段描述检索器如何接收查询、返回候选结果，以及多个检索器如何被统一编排。

检索阶段可以支持单路检索，也可以支持多个检索器并行执行，再把结果交给后续阶段处理。

`retrieval` 不实现具体向量数据库、搜索引擎或召回服务。

### 检索后

检索后负责把候选结果整理为稳定的最终检索结果。

该阶段描述候选结果如何经过融合、去重、过滤、重排、截断和结果组装等处理后返回给调用方。

`retrieval` 可以提供分数阈值过滤、结果去重、Top K 截取等无外部依赖、无业务语义的通用处理能力。

`retrieval` 只定义 rerank 协议，不实现具体 rerank 模型。

## 流程编排

`retrieval` 的流程编排只负责组织阶段调用，不负责阶段内部的具体业务或外部能力实现。

基础流程为：

```text
原始检索请求
  → 检索前处理器
  → 检索器
  → 检索后处理器
  → 检索结果
```

当存在多个检索前处理器或检索后处理器时，按照声明顺序串行执行。

当存在多个检索器时，可以按统一协议并行执行，并将候选结果交给检索后阶段继续处理。

## 与 adapters 的边界

`retrieval` 负责检索领域的通用协议、通用编排和无外部依赖的基础处理能力。

`adapters` 负责把外部生态能力适配为 `retrieval` 可以消费的协议实现。

例如，LangChain 查询改写、向量数据库检索器、搜索引擎检索器、rerank 服务适配、Embedding 模型调用等能力属于 `adapters` 或外部实现，不属于 `retrieval`。

## 不负责

`retrieval` 不实现具体向量数据库、搜索引擎或召回服务。

`retrieval` 不实现具体 rerank 模型。

`retrieval` 不承载业务权限逻辑。

`retrieval` 不决定检索结果如何生成最终业务响应。

`retrieval` 不提供绑定具体外部生态或业务场景的完整流程预设。
